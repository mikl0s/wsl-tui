  <action>
    Create `CLAUDE.md` at the repo root. This is the primary reference document for any agent working in this codebase. Read the actual source files from Plans 01-03 to document what was ACTUALLY built (not what was planned).

    Structure the document with these sections:

    **Project Overview:**
    - WSL TUI: Rust-based TUI + Web UI for managing WSL2 on Windows 11
    - Cargo workspace monorepo: wsl-core (shared lib) + wsl-tui (TUI binary) + wsl-web (Web binary)
    - MIT License, Rust 1.88 MSRV, edition 2021

    **Architecture:**
    - Workspace structure diagram (directory tree of actual files)
    - Dependency flow: wsl-tui -> wsl-core, wsl-web -> wsl-core
    - Module map for wsl-core: config, error, storage (libsql + json), wsl (executor), plugin (trait + registry)
    - Module map for wsl-tui: main (entry + event loop), app (state), ui (welcome screen)

    **Coding Standards:**
    - Error handling: `thiserror` for typed errors in wsl-core (`CoreError`), `anyhow` for ergonomic propagation in wsl-tui and wsl-web
    - Unwrap policy: NEVER use `.unwrap()` in production code; `.expect("reason")` only in initialization where failure is unrecoverable; `?` operator everywhere else
    - Visibility: Default to `pub(crate)` for internal items, `pub` only for cross-crate API
    - Doc comments: Required on all public types, traits, and functions (`///` with examples where non-trivial)
    - Imports: Group by std, external crates, internal modules; separated by blank lines
    - Tests: Unit tests in the same file (`#[cfg(test)] mod tests`), integration tests in `tests/` directory at crate root

    **Testing Requirements** (per locked decision):
    - Unit tests required for every public function
    - Integration tests for cross-crate boundaries
    - `cargo test --workspace` must pass with zero failures
    - `cargo clippy --workspace -- -D warnings` must pass with zero warnings
    - No `.unwrap()` in tests except when the test is specifically testing the happy path and failure would indicate a test bug

    **Windows Platform Requirements:**
    - `.cargo/config.toml` sets `/STACK:8000000` for libsql (do not remove)
    - `KeyEventKind::Press` filter required on ALL key event handlers (crossterm fires Press+Release on Windows)
    - WSL output encoding: check `WSL_UTF8` env var at runtime — UTF-8 when set, UTF-16LE otherwise
    - Always strip trailing null bytes from wsl.exe output

    **Performance Targets:**
    - DX-05: Startup time under 500ms to first render
    - DX-06: Idle memory usage under 50MB
    - DX-07: Binary size under 30MB (release profile configured with opt-level z, LTO, strip)

    **Async Runtime:**
    - tokio is the runtime (required by libsql)
    - `#[tokio::main]` on entry points
    - `#[tokio::test]` on async test functions
    - `async_trait` used for `StorageBackend` (dyn object safety requires it)

    **Configuration:**
    - Config file: `~/.wsl-tui/config.toml`
    - Auto-created on first run with fully commented defaults
    - Env var overrides: `WSL_TUI_*` prefix (e.g., `WSL_TUI_STORAGE=json`)
    - Config dir: `~/.wsl-tui/` (resolved via `dirs::home_dir()`)

    **Per-crate docs:**
    - Link to wsl-core/CLAUDE.md, wsl-tui/CLAUDE.md, wsl-web/CLAUDE.md

    Verify `cargo clippy --workspace -- -D warnings` still passes (CLAUDE.md doesn't affect code, but verify no regressions from any file changes).
