    - `WslExecutor` struct (stateless for Phase 1 — may hold config in later phases)
    - `WslExecutor::new() -> Self`
    - `fn is_utf8_mode() -> bool` — checks `std::env::var_os("WSL_UTF8")` returns `Some` with value "1"
    - `fn decode_output(raw: &[u8]) -> String`:
      - If `is_utf8_mode()`: `String::from_utf8_lossy(raw).into_owned()`
      - Else: `encoding_rs::UTF_16LE.decode(raw).0.into_owned()`
      - Always: `.trim_end_matches('\0').trim().to_string()` — strip null bytes per research
    - `fn run(&self, args: &[&str]) -> Result<String, CoreError>`:
      - `Command::new("wsl.exe").args(args).output()`
      - Check `output.status.success()`; if not, return `CoreError::WslFailed` with stderr
      - Decode stdout via `decode_output`
    - `fn list_verbose(&self) -> Result<String, CoreError>`:
      - Calls `self.run(&["--list", "--verbose"])`
    - Write tests:
      - `test_decode_output_utf8()` — with WSL_UTF8=1 env, decode valid UTF-8 bytes
      - `test_decode_output_utf16le()` — without WSL_UTF8, decode known UTF-16LE bytes (include test vector)
      - `test_null_byte_stripping()` — output with trailing \0 is cleaned
      - `test_is_utf8_mode()` — test with and without WSL_UTF8 env var (use `std::env::set_var` in test, noting it's unsafe in multi-threaded tests — run sequentially or use serial_test)

    NOTE: Do NOT run `wsl.exe` in tests — mock the Command output or test the decode/parse functions directly. WSL may not be available in CI.

    - `fn dismiss_welcome(&mut self)` — sets `show_welcome = false`

    Create `wsl-tui/src/ui/mod.rs`:
    - `pub mod welcome;`
    - `pub fn render(app: &App, frame: &mut ratatui::Frame)`:
      - If `app.show_welcome` → render welcome screen
      - Else → render placeholder main screen ("WSL TUI v0.1.0 — Press q to quit")

    Create `wsl-tui/src/ui/welcome.rs`:
    - Render a polished welcome screen (per locked decision: "should feel polished — first impression"):
      - Centered block with title "Welcome to WSL TUI"
      - Key info lines:
        - "Configuration: ~/.wsl-tui/config.toml"
        - "A fully-commented config file has been created for you."
        - "Customize settings by editing the config file."
      - Footer: "Press any key to continue..."
    - Use `ratatui::widgets::Paragraph` inside a `Block` with `Borders::ALL`
    - Layout centered both vertically and horizontally using `Layout` constraints

    Update `wsl-tui/src/main.rs`:
    - `#[tokio::main] async fn main() -> anyhow::Result<()>`:
      1. Load config: `let config = Config::load()?;`
      2. Create app: `let mut app = App::new(&config);`
      3. Init terminal: `let mut terminal = ratatui::init();`
      4. Run event loop: `let result = run_app(&mut terminal, &mut app).await;`
      5. Restore terminal: `ratatui::restore();`
      6. Return result
    - `async fn run_app(terminal, app) -> anyhow::Result<()>`:
      - Loop while `app.running`:
        1. `terminal.draw(|frame| ui::render(&app, frame))?;`
        2. Read event: `crossterm::event::read()?`
        3. If `Event::Key(key)` AND `key.kind == KeyEventKind::Press`:
           - If `app.show_welcome` → `app.dismiss_welcome()` (any key dismisses)
           - Else match `key.code`:
             - `KeyCode::Char('q')` → `app.quit()`
             - `_` → ignore (placeholder for Phase 2 keybindings)

    NOTE on sync vs async event reading: For the Phase 1 skeleton, use synchronous `crossterm::event::read()` which is simpler and sufficient. Phase 2 will upgrade to `EventStream` with `tokio::select!` when async background tasks are needed. The `KeyEventKind::Press` filter is the critical Windows fix regardless of sync/async.

    Write tests:

    - `test_app_new_not_first_run()` — App with first_run=false has show_welcome=false
    - `test_app_quit()` — calling quit() sets running=false
    - `test_app_dismiss_welcome()` — calling dismiss_welcome() sets show_welcome=false
