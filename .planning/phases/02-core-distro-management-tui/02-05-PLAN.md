**Install flow (2 phases per locked decision):**

Phase A — Distro picker:
- When user presses the install key (Claude's discretion: could be a dedicated key like `I` or via an action menu):
  1. Call `app.executor.list_online()` to get available distros
  2. Set `app.modal = ModalState::InstallPicker { online_distros, list_state: ListState::default().with_selected(Some(0)) }`
  3. Render a selection list of online distros (name + friendly name)
  4. j/k to navigate, Enter to select and start install, Esc to cancel

Phase B — Progress modal:
Per locked decision: modal overlay with progress bar, blocks distro list until complete.

1. When user selects a distro from the picker:
   - Set `app.modal = ModalState::InstallProgress { distro_name, step: "Starting install...".into(), percent: 0, completed: false }`
   - Spawn the install command in a `tokio::task::spawn_blocking` (because `wsl --install` is a long-running blocking process)
   - Use a `tokio::sync::mpsc::channel` to send progress updates from the blocking task to the main loop
   - In the `tokio::select!` loop, add a branch for the install progress channel receiver

2. Progress tracking (per Pitfall 7 in research — no machine-readable progress from `wsl --install`):
   - The blocking task runs `std::process::Command::new("wsl.exe").args(["--install", &name])` and captures stdout line by line using `BufReader::new(child.stdout.take())`
   - Map output lines to coarse step labels:
     - Initial: "Downloading..." (0-30%)
     - "Installing..." (30-70%)
     - "Configuring..." (70-90%)
     - On process exit: "Complete" (100%)
   - If keyword detection is unreliable, use a time-based approach: increment percent by 1 every 2 seconds, capped at 90%. Jump to 100% on process exit.
   - Send `(step: String, percent: u16)` tuples through the channel

3. The main loop receives progress updates and sets `app.modal.step` and `app.modal.percent`
4. When completed: set `completed: true`, allow any key to dismiss the modal, call `app.refresh_distros()`

**Wire into event handler:**

When `ModalState::ExportInput`:
- `Char(c)` → insert char at cursor position in `path`, advance cursor
- `Backspace` → remove char before cursor, decrement cursor
- `Left` → decrement cursor (clamp to 0)
- `Right` → increment cursor (clamp to path.len())
- `Enter` → execute export:
  1. Call `app.executor.export_distro(&distro_name, &path)`
  2. On success: `app.modal = ModalState::None`, optionally show brief success message (Claude's discretion)
  3. On error: show the error in the modal or set `app.modal = ModalState::None` and continue
- `Esc` → `app.modal = ModalState::None`

When `ModalState::ImportInput`:
- Same character/cursor handling as export, but applied to the `active_field`
- `Tab` → cycle `active_field` (0 → 1 → 2 → 0)
- `Enter` → execute import:
  1. Validate: name, install_dir, and tar_path must all be non-empty
  2. Call `app.executor.import_distro(&name, &install_dir, &tar_path)`
  3. On success: `app.modal = ModalState::None`, call `app.refresh_distros()`
  4. On error: show error or dismiss
- `Esc` → `app.modal = ModalState::None`
