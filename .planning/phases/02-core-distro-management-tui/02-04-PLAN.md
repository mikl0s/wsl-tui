**Shell attach (in main.rs):**

Per locked decisions:
- Instant swap: TUI clears immediately, no transition message
- Auto-start: Enter on stopped distro starts it first
- Exit shell normally: type 'exit' or Ctrl+D
- Full restore on return: exact scroll position, selection, and view state preserved

Implement in the event handler when `Action::AttachShell` is triggered:

```rust
// Shell attach per locked decision: instant swap, auto-start, full restore
if let Some(distro) = app.selected_distro().cloned() {
    // Auto-start if stopped
    if distro.state == DistroState::Stopped {
        let _ = app.executor.start_distro(&distro.name);
    }

    // Instant swap: suspend TUI
    ratatui::restore();

    // Drop into shell — inherits stdin/stdout/stderr
    let _ = std::process::Command::new("wsl.exe")
        .args(["-d", &distro.name])
        .status();

    // Full restore — re-init TUI, reinstall panic hook
    // Note: terminal handle must be reassigned
    *terminal = ratatui::init();

    // State is already preserved in app (list_state, selection, view)
    // Refresh distro list to pick up state changes from the shell session
    let _ = app.refresh_distros();
}
```

CRITICAL: The `terminal` variable must be accessible and mutable at the point of shell attach. The event handler needs to be restructured so that `terminal` is passed as `&mut` or the attach logic lives in `run_app` where `terminal` is already mutable.

Pattern: In `run_app`, after receiving an `AttachShell` action, break out of the event match, execute the shell attach, then continue the loop. Do NOT spawn this in a tokio task — it must run synchronously on the main thread with inherited stdio.
