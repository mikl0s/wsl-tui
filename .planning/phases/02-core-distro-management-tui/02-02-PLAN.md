**Step 2: Create keybindings module**

Create `wsl-tui/src/keybindings.rs`:

1. `pub fn parse_key_str(s: &str) -> Option<(KeyCode, KeyModifiers)>` ‚Äî parse a string like "j", "enter", "ctrl+d", "?", "esc", "tab", "backspace", "f1" into a `(KeyCode, KeyModifiers)` pair.

Mapping:
- Single char (len 1): `KeyCode::Char(c)`, `KeyModifiers::NONE`
- "ctrl+<c>": `KeyCode::Char(c)`, `KeyModifiers::CONTROL`
- "alt+<c>": `KeyCode::Char(c)`, `KeyModifiers::ALT`
- "enter": `KeyCode::Enter`, `KeyModifiers::NONE`
- "esc" / "escape": `KeyCode::Esc`, `KeyModifiers::NONE`
- "tab": `KeyCode::Tab`, `KeyModifiers::NONE`
- "backspace": `KeyCode::Backspace`, `KeyModifiers::NONE`
- "up"/"down"/"left"/"right": arrow KeyCodes
- "f1".."f12": `KeyCode::F(n)`
- "space": `KeyCode::Char(' ')`
- "1".."9": `KeyCode::Char('1')` etc.

2. `pub struct KeyBindings` with parsed `(KeyCode, KeyModifiers)` fields for each action. Constructor `pub fn from_config(config: &Config) -> Self` that calls `parse_key_str` for each keybinding string. Use `expect("keybinding '...' is not a valid key")` for truly invalid keys (config validation at startup, not runtime).

3. `pub fn matches(&self, key: &KeyEvent, action: KeyAction) -> bool` ‚Äî compare `key.code == action_code && key.modifiers.contains(action_modifiers)`.

4. `pub enum KeyAction` ‚Äî one variant per action: `Quit, Help, Filter, Up, Down, Left, Right, Attach, Start, Stop, SetDefault, Remove, Export, Import`.

Add `pub mod keybindings;` to `wsl-tui/src/main.rs`.
